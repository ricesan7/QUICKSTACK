その在庫データ、本当に信じられますか？

在庫と原価がズレる

→ 移動平均で自動計算。入庫のたびに原価を再計算し、粗利まで一貫して扱えます。

セット販売で誤出庫

→ 最大出庫可能数＆不足強調。構成品在庫から、あと何セット作れるかを瞬時に判断します。

後から数字が合わない

→ 取消（REVERSAL）と監査ログ。削除せず履歴を残しながら修正でき、原因追跡まで行えます。

現場と経営をつなぐ機能

在庫・原価管理

  • 商品マスタ（画像 / SKU / カテゴリ / 保管場所）
  • 最小在庫アラート（画面上で強調表示）
  • 移動平均原価（入庫で自動更新）
  • 在庫金額・売上・原価・粗利の自動計算

Bundle（セット販売）

  • 複数SKUをセット化（Bundle / BundleItem）
  • 数量別価格（BundlePrice）で価格帯を管理
  • 最大出庫可能セット数を算出
  • セット出庫で構成品在庫を一括減算（原価配分）

作業（Work）

  • QRスキャンまたは手動選択で即フォーム遷移
  • 入庫：数量・仕入単価→移動平均単価を自動算出
  • 出庫：販売単価・販売先→売上/原価/粗利を自動算出
  • 棚卸調整：実在庫入力で差異を自動反映

統制（ガバナンス）

  • 取消（REVERSAL）：削除せず反対仕訳で整合性を維持
  • 監査ログ（Audit Log）：実行者・対象・差分を記録
  • 整合性チェック：在庫数/単価/金額の不整合を検出
  • 棚卸（Stocktake）：下書き→ロック→確定で期末棚卸に対応

レポート（Dashboard）

  • 総在庫数・在庫総額・アラート数を即表示
  • 入荷/出荷/販売金額/粗利を集計
  • 販売ランキング（上位10）・販売先別集計
  • セット販売ランキング（上位10）

入出力（CSV / QR）

  • 商品CSVインポート / 在庫CSVエクスポート
  • 入庫CSVインポート / 履歴CSVエクスポート
  • 棚卸結果CSVエクスポート
  • QRコード生成・ダウンロード（PNG / PDF）

FAQ

スマホだけで運用できますか？
はい。QRスキャン入出庫はスマホ運用を前提に設計しています。PCは管理や確認用途にも併用可能です。
取消（REVERSAL）とは何ですか？
誤操作を「削除」せず、反対仕訳として打ち消す方式です。履歴を保持したまま、在庫・原価・売上の整合性を維持できます。
棚卸（Stocktake）は何ができますか？
下書き→ロック→確定のステータスで棚卸を管理し、評価基準日時に基づくスナップショットを作成します。期末棚卸にも対応します。
