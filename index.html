今すぐ始める（Stripeへ）

その在庫データ、本当に信じられますか？

在庫と原価がズレる

→ 移動平均で自動計算。入庫のたびに単価を更新し、粗利を自動計算します。

セット販売で誤出庫

→ Bundle（セット）出庫。構成品の在庫を一括で減算し、原価配分も自動計算します。

修正が履歴に残らない

→ 取消（REVERSAL）。削除ではなく反対仕訳で打ち消し、履歴を維持します。

現場と経営をつなぐ機能

在庫・原価管理

  • 商品マスタ（画像 / SKU / カテゴリ / 保管場所 / 最小在庫）
  • リアルタイム在庫更新
  • 移動平均原価（入庫時に自動更新）
  • 在庫金額（移動平均単価×在庫数）を自動計算
  • 在庫アラート（最小在庫割れの検知）

Work（入出庫・棚卸調整）

  • QRスキャンまたは手動選択で商品を即選択
  • 入庫：数量・仕入単価入力 → 単価を自動再計算
  • 出庫：数量・販売単価・販売先 → 売上/原価/粗利を自動計算
  • 棚卸調整：実在庫数入力 → 差異を自動反映
  • 履歴管理：検索/フィルター/ソート、編集・取消（REVERSAL）

Bundle（セット販売）

  • 複数SKUをセット化（Bundle / 構成品数量 / 並び順）
  • 個数別の価格設定（BundlePrice）
  • セット出庫：構成品在庫を一括減算
  • 構成品ごとの原価配分・粗利の自動計算

レポート（Dashboard）

  • 統計：商品数/在庫数/アラート数/在庫総額
  • 入荷数/出荷数/販売金額/粗利（期間集計）
  • 販売ランキング（商品別）・販売履歴（日時/数量/金額）
  • 販売先ランキング（販売先別の売上集計）
  • セット販売ランキング（セット別の売上・粗利）

棚卸（Stocktake）

  • 棚卸セッション（下書き→ロック→確定→取消）
  • 帳簿在庫と実地棚卸の差異を自動計算
  • 評価単価・評価額の計算、スナップショット保存
  • 棚卸結果CSVエクスポート

CSV / 整合性 / 監査

  • 商品マスタCSVインポート / 在庫CSVエクスポート
  • 入庫CSVインポート / 履歴CSVエクスポート
  • 棚卸結果CSVエクスポート
  • データ整合性チェック（在庫数・単価の検証）
  • 監査ログ（操作履歴・変更差分）

FAQ

スマホだけで運用できますか？
はい。QRスキャン入出庫はスマホ運用を前提に設計しています。PCは管理や確認用途にも併用可能です。
取消（REVERSAL）とは何ですか？
誤操作を削除せず、反対仕訳として打ち消す方式です。履歴を維持しながら、在庫・原価の整合性を保てます。
問い合わせはどこからできますか？
